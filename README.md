# programtest_2026

## 概要

プログラム課題として、以下 3 問の CLI アプリケーションを Java で実装しました。

- 課題1：すごろくのゴールに到達できる最小のサイコロ回数を求める
- 課題2：1次元リバーシの棋譜から、最終的な黒石と白石の個数を求める
- 課題3：整数の各桁を並べ替えて、先頭が 0 にならない最小値を求める

いずれも、標準入力を受け取り、標準出力に結果を出力する CLI アプリケーションとして実装しています。

## 各課題の実装プログラム

### 課題1

#### ＜実装方針＞

全長 `A` マスのすごろくについて、ゴールに到達するための最小サイコロ回数を求めるプログラムです。

この課題では、スタート地点に最初から駒が置かれているため、実際に進む必要があるのは `A - 1` マスです。  
1回のサイコロで最大 6 マス進めるので、`(A - 1) / 6` を切り上げることで最小回数を求めています。

実装では、標準入力から整数 `A` を受け取り、`Math.ceil((A - 1) / 6.0)` で必要回数を算出して出力しています。

#### ＜実行結果＞

| 入力値 | 期待する出力 | 実際の出力 |
|---|---:|---:|
| `2` | `1` | `1` |
| `10` | `2` | `2` |
| `13` | `2` | `2` |
| `100` | `17` | `17` |

### 課題2

#### ＜実装方針＞

1次元リバーシの棋譜 `S` から、最終的な黒石と白石の個数を求めるプログラムです。

この課題では、盤面全体をそのまま配列で持つのではなく、**連続する同色の石を 1 つのブロックとして管理する**方法で実装しています。  
たとえば `B B B W W B` という状態は、`[(B,3), (W,2), (B,1)]` のようなブロック列として扱います。

石を置けるのは常に左端または右端だけなので、影響するのも端のブロックだけです。  
そのため、`ArrayDeque` を使って左右端を効率よく更新する構成にしています。

実装では、以下の流れで処理しています。

- 初期配置として「黒 1 個、白 1 個」を入れる
- 棋譜を 1 文字ずつ読み取る
- 手番に応じて黒石・白石を決める（奇数手⇒黒、偶数手⇒白）
- `L` の場合は左端、`R` の場合は右端に石を置く
- 同色ならブロックを伸ばす
- 異色で反対側に同色ブロックがある場合は、端ブロックを反転して隣の同色ブロックにまとめる
- 異色で同色石が存在しない場合は、反転せず新しいブロックを追加する
- 最後に全ブロックを集計して黒石数と白石数を出力する

#### ＜実行結果＞

| 入力値 | 期待する出力 | 実際の出力 |
|---|---:|---:|
| `RRLL` | `0 6` | `0 6` |
| `LLRLRLR` | `3 6` | `3 6` |

### 課題3

#### ＜実装方針＞

10 進数で表記された整数 `x` の各桁を並べ替えた整数のうち、先頭が 0 にならないものの中で最小の値を求めるプログラムです。

この課題では、まず入力値を文字列として受け取り、各桁を `char[]` に変換して昇順ソートしています。  
ただし、単純にソートすると先頭が `0` になる場合があるため、その場合は**最初に現れる非 0 の数字を先頭と入れ替える**ことで、条件を満たす最小値を作っています。

実装では、以下の流れで処理しています。

- 入力を文字列として受け取る
- `char[]` に変換する
- `Arrays.sort` で昇順に並べ替える
- 先頭が `0` の場合は、最初の非 0 の桁を探して先頭と交換する
- 並べ替え後の文字列を出力する

#### ＜実行結果＞

| 入力値 | 期待する出力 | 実際の出力 |
|---|---:|---:|
| `201` | `102` | `102` |
| `12212122212222` | `11112222222222` | `11112222222222` |
